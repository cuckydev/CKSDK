Detailed Description

As the GPU runs separately to the CPU, CKSDK provides an asynchronous command queue for DMAs between the CPU and GPU, which includes DMAing GP0 packets to the GPU (ordering tables) and DMAing images to VRAM.

The command queue is implemented as a basic FIFO queue.

The default queue size is 16. If this is exhausted, the CPU will spin until a slot is free.

void CKSDK::GPU::Queue_OrderingTableDMA ( const Tag &  buffer)

DMA ordering table to GPU.

Parameters
bufferThe beginning of the ordering table

The simplest way to get commands from the CPU to the GPU on PS1 is often to DMA an ordering table to the GPU.

Each packet begins with a Tag which points to the next packet, and ends with a null terminator.

Note
This is an asynchronous command, so the given pointer must be valid until the queue is empty
